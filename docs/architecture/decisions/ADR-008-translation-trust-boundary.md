# ADR-008: Translation trust boundary

- Status: Accepted
- Date: 2026-06-22

**In one minute:** we want notes a reader can study in their own language without weakening the provenance guarantee that is the whole point of the tool. So we draw a single line: **translation happens at the reading/export layer, never at extraction, and never inside the anchored excerpt.** Reader prose and the invented `##`/`###` headings are written by the synthesis call (`export.study_document`), which works only from already-anchored notes — so writing that prose in the target language is **faithfulness-neutral**: it is a re-representation that still cites real `[s-N]` note ids (bracketed ascii tokens that survive translation), the same stance ADR-007 took that *a re-representation is not the anchor*. The verbatim **excerpt stays the sole anchored evidence and is never translated in place** — the §7 anchor checks (`anchor_gate`, `check_traceability`) read `excerpt` only; a translation is a separate additive `evidence[].excerpt_translation` gloss, a clearly-marked reading aid that never enters a §7 check and renders *beneath* the original quote for cited excerpts, never replacing it. We capture `detected_language` per page at extraction (free — the model already reads the page) and carry it through the gate and compose like the `generation` settings (#98). A **semiautomatic, advisory gate** offers confirm-then-translate when the detected source language differs from the resolved preferred language (flag > config > OS-locale > `en`); it never blocks and is skippable for non-interactive runs. We accept **any** language with a soft warning rather than maintaining a hard allowlist (mirroring the permissive `effort` handling). And we name the **measurement blind spot** plainly: the floor verifies the *original excerpt* is present — it **cannot** see whether *translated prose* is faithful to the source; that fidelity is the human spot-check rubric's job, not the floor's. Schema additions are **additive — no `schema_version` bump, no migration** — per the #98 precedent that per-page notes are regenerable app output, not adopter state.

## Context

The tool produces notes from sources whose language need not match the reader's. A maintainer reading Egyptological scholarship in German, or a reader who wants the study document in English from a French source, has no path today: every produced surface is in the source language. We want to add translation — but translation is exactly the kind of feature that can quietly dissolve the provenance guarantee the tool exists to defend. A quote that has been translated is no longer the quote; a paraphrase translated through a model is a paraphrase of a paraphrase. The question is not *whether* to translate but *where translation is allowed to touch*, so that adding it never weakens what §7's mechanical checks prove.

There are three places translation could plausibly live, and only one of them is safe:

- **At extraction** (translate the page, or translate the notes as they are written). This is the tempting one and it is wrong. It would translate an *intermediate* — the page text or the freshly-written statement/evidence records — and so perturb the mechanical compose stages that read that text byte-for-byte: dedup blocking compares `statement.text` and the verbatim `excerpt` (`METHODOLOGY.md` §6 compose), and a translated intermediate breaks the string adjacency those stages depend on. Worse, it puts the quality risk on the *one* call the deterministic floor-score can't fully police (the synthesis), at the exact stage where the anchor must still be verbatim against the source page.
- **Inside the anchored excerpt** (translate the quote in place). This breaks the anchor outright. `anchor_gate` and `check_traceability` resolve `excerpt` by verbatim normalized-substring containment against the source page (`normalize.py`, §6); a translated excerpt resolves against nothing and is dropped, or — worse — is silently kept as evidence that no longer quotes the source.
- **At the reading/export layer** (translate the synthesized reader prose, leaving every anchored record native). This is the safe seam, and it is where this ADR puts it.

This records the design reviewed in the translation-trust-boundary session. It stands on two accepted decisions: ADR-006 (the page-identity distinction the per-page `detected_language` capture rides on) and ADR-007 (the floor is an instrument that verifies the anchor, and a re-representation is not the anchor).

## Decision

**Translation lives at the reading/export layer, never at extraction.** The reader prose and the invented `##`/`###` study-document headings are written by the synthesis call (`export.study_document`), which by construction works **only from already-anchored notes** — it never re-reads the source page and never touches an `excerpt`. Writing that prose in the target language is therefore **faithfulness-neutral**: it is a re-representation of notes that are *already* anchored, and it still cites the real `[s-N]` note ids — bracketed ascii tokens that pass through a translation model intact, so the citation graph survives. This is ADR-007's stance applied to language: *a re-representation is not the anchor*, so re-representing it in another language changes nothing the floor cares about. **Folding translation into extraction is rejected** for the three reasons above — it translates an intermediate, perturbs the mechanical compose stages (dedup blocking reads `statement.text`/`excerpt`), and loads quality risk onto the call the floor can least police. Extraction stays native-language, full stop.

**The verbatim excerpt is the sole anchored evidence and is never translated in place.** The anchor is the quote, and the quote is the source's own words. `anchor_gate` and `check_traceability` read `excerpt` and nothing else (§7.2). Any translation of a quote is a **separate, additive field** — `evidence[].excerpt_translation` — a clearly-marked reading aid that **never enters any §7 check**. It is produced and rendered only for the *cited* excerpts (the ones a study document actually surfaces), shown beneath the original and visually distinct, and it **never replaces** the verbatim quote. The anchor stays the source's words; the gloss sits beside it as help, not as evidence.

**`detected_language` is captured per page at extraction.** The extractor already reads every page, so naming the page's language costs nothing extra. It is stored per page (`page-NNNN.notes.yaml`) and carried through the anchor gate and compose exactly the way the `generation` settings are (#98) — the same regenerable, per-page provenance seam ADR-007 leans on. Per-page capture (not a single run-level stamp) is also what lets a mixed-language document be seen as mixed, the same reasoning that made `generation` per-page.

**The translate-offer gate is semiautomatic and advisory.** When a page's `detected_language` differs from the resolved **preferred language**, the tool offers *confirm-then-translate* — it never translates silently and **never blocks** the pipeline. It is skippable for non-interactive runs (it simply does not fire). Preferred language resolves in order: **command flag > config > OS-locale > `en`**. This is the §7.6 *software-assists / human-decides* posture (the same frame ADR-007 invokes) applied to translation: the tool surfaces the opportunity, the human chooses.

**Any language is accepted with a soft warning; there is no hard allowlist.** We deliberately do not maintain a supported-language list. It would drift as the underlying models gain and lose competence, and a hard gate would refuse languages the model can in fact handle. We accept any target, emit a soft warning the user can act on, and let the human judge the result — mirroring the permissive `effort` handling, which likewise soft-warns rather than enumerating an allowlist that goes stale.

**The measurement blind spot, stated plainly.** The deterministic floor verifies that the **original excerpt** is present and anchored on its source page. It **cannot** see whether **translated prose** — neither the reader synthesis nor the `excerpt_translation` gloss — faithfully reflects the source, because translation fidelity is not a verbatim-containment fact and the floor measures only verbatim-containment facts. Translated-prose fidelity is the **human spot-check rubric's** job, not the floor's. This is ADR-007's instrument boundary inherited exactly: the floor proves the anchor is real; it does not — and here *cannot* — grade a re-representation, and saying otherwise would be the laundering ADR-007 forbids.

**Schema additions are additive — no `schema_version` bump, no migration.** Two fields are added: `evidence[].excerpt_translation` and a top-level `detected_language`. Both are optional; notes produced before them simply read as absent, never as a contradiction. Per the #98 precedent (carried in ADR-007's consequences), per-page notes are **regenerable application output in a user work dir, not adopter methodology state**, so the kit's *surface-change → schema-version-bump-and-migration* rule (`.pkit/rules/core.md` rule 7; COR-010) does not bind here — re-running extraction reproduces the fields.

## Consequences

- **Translation can never break an anchor, by construction.** Because the only translated surfaces are the synthesized reader prose and an additive gloss field — and the verbatim `excerpt` is left untouched — there is no path by which adding translation degrades a §7 check. A future change that translates `excerpt` *in place*, or translates the page before extraction, reopens this ADR and must say so explicitly (the supersession gesture); both are the failure modes this record exists to forbid.
- **`excerpt_translation` must stay out of every §7 check, forever.** Its safety is that it is a reading aid the validators never read. A future change that lets `anchor_gate` or `check_traceability` consider `excerpt_translation` — even "just to flag a mismatch" — has turned a gloss into evidence and broken the boundary. The verbatim quote is the only thing the anchor resolves against.
- **The synthesis call must work only from anchored notes — translation does not change that.** Translating at export is faithfulness-neutral *precisely because* `export.study_document` never re-reads the source. A future export path that reaches back to the page to "improve" a translation has dissolved the property this ADR depends on; reader-layer translation is sound only while the reader layer reads notes, not sources.
- **The floor's blind spot is now a documented contract, not an oversight.** Anyone reading a floor score on a translated run must understand the score speaks to the *original* excerpts being present, and says nothing about translation quality. A future maintainer who wants a translation-fidelity number is proposing a new instrument (and inherits ADR-007's whole instrument discipline — fingerprint, fact-vs-measurement separation, no Goodharting); they are not extending the floor.
- **The detect/offer/translate path stays advisory and isolated.** The gate never blocks and is skippable; the detected-language capture is free and per-page. A non-interactive or offline run produces native notes exactly as today. This keeps translation out of the critical path and out of the pipeline's network-posture story (Invariant 5): the offer is reader-layer, the same side of the line as export.
- **`detected_language` is a per-page provenance fact, like `generation`.** It rides the same seam ADR-006 and ADR-007 established — encoded per page, recovered from on-disk artifacts, never carried as hidden pipeline state. A mixed-language document is therefore *visible* as mixed, never collapsed to a single run-level label.
- **The additive-schema judgement holds only while these stay regenerable app output.** The no-bump/no-migration call rests on the #98 precedent that per-page notes are user-work-dir output, not adopter methodology state. If a future change made either field load-bearing for *adopter* state (e.g. committed into the kit surface), that precedent no longer covers it and the migration rule reapplies.
- **No hard language allowlist means the warning is the only guardrail.** We accept this: the soft warning plus human judgement replaces an allowlist that would drift with the models. A reviewer who later wants to *refuse* a language is adding a gate where this ADR chose an advisory warning, and is reopening the permissive-handling decision.
